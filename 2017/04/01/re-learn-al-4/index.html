Algorithms:Merge Sort && Quick Sort

发表于 2017-04-01 | |

归并排序

简单特性

  • 使用了分治的思想
  • 也适合大型的数据量
  • 又一个 $ O(nlog(n)) $ 级别的排序算法

首先是比较重要的 merge操作：

当调用 merge 操作的时候，我们会保证，传入的两个数组都是有序的数组。从两个数组的开头分别用两个指针进行便利，选择两者之间较小（大）的进入对应位置，除此之外还要考虑到两个数组大小不一样的情况，针对这四种情况进行判断移动浮标。

自下而上的归并排序

merge 形成了一个向下递归的操作，通过分治法，数组被最小化分到只有一个元素（最小情况的有序），然后两两排序合并，然后递归不断的向上返回获得整个的排序好的数组。

Tips: 复杂度的证明

这个复杂度的证明书上使用了数学归纳法，但是在第二节我们已经证明过了主定理法和递归树分析法：

这里的表达式是 : $ T(N) = 2T(N/2) + N $

$ n^{log_2^2} = n $ 所以 $ O(T(N)) = nlog^n $

Tips: 对于长度为 n 的任意数组，自定向下的归并排序最多需要访问数组 $ 6nlg(n) $ 次。

每次归并最多需要访问数组 $ 6N $ 次，$ 2N $ 用来复制，$2N$ 用来将排序好的元素还原，另外最多因为比较方粉 $2N$ 次数组。

优化方法

  • 对小规模的数组使用插入排序：
  • 每次合并之前判断是否数组已经有序，如果有序就不用走 merge 了：

判断是否已经合并的方法是判断前一半数组的最大值和后一半大小的最小值的关系，即 list[mid] 和 list[mid + 1] 两者之间的关系。

  • 不将元素复制到辅助数组，每次都切换数组和辅助数组的关系，来交叉排序。

自下而上的归并排序

自下而上而上的归并排序比较适合处理链表组织的数据结构，这样只需要重新组织链表就可以了，不需要在另外的开辟空间。
