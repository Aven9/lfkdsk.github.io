选择排序

简单特性 ：

  • 双层循环
  • 每次确定一位的数据
  • $ O(n^2) $

选择排序是通过每次确定一个（最小或最大的位）来进行排序的算法，从定义上明显能看出排序的过程和输入的规模有关，但是和数组本身的顺序无关，因为没有运用数组本身的顺序，所以无论是接近有序的数组，还是完全乱序的数组，只要规模一样，复杂度都一样。

插入排序

简单特性：

  • 排序的游标 $ i $ 永远保证 $ i $ 左侧的数组是有序的
  • 每次游标 $ i $ 发现一个新的元素，会向左侧的数组进行插入
  • $ O(n^2) $

插入排序是另一个 $ O(n^2) $ 的排序算法，排序的游标 $ i $ 永远保证 $ i $ 左侧的数组是有序的，每次游标 $ i $ 发现一个新的元素，会向左侧的数组进行插入，如果比最后一个元素大，我们就可以直接 break 不管了。

Tips 倒置对

数组中的逆序数对被称为倒置对，选择排序的没次交换都会进行一个倒置对的颠倒。

这样子，本身部分有序或者是本身包含的逆序对少的数组使用选择排序速度可以加快。

还有如果把交换改成向右移动，数组的访问次数就能减半。

希尔排序

第一个不平凡的排序算法。

简单特性：

  • 改良版的插入排序
  • 分成多组，组序列之间进行排序
  • 首个复杂度进入 $ O(Nlog^2(N)) $ 的算法 最坏的情况和 $ O(N^{3/2}) $ 成正比

之前一部分排序算法速度缓慢的原因是一次的移动只能是相邻的两个元素之间进行的。希尔排序使用多个个序列之间进比较的方式，能将一个数移到很远的地方。

Tips: 对 g 排序后的序列，对 h 排序依然保证顺序。这是 Shell Sort 能使用多段排序的理论基础。

ShellSort 的过程可以类比 InsertSort 的操作，最后一段 h = 1 的排序就是一个插入排序，但是前面几段的排序已将减少了很多的逆序数对。

在数组规模一定的时候可以优先选择希尔排序，代码量少，而且还有不错的效率。

Kruth Shuffle

里面提到的一个洗牌的算法，时间的复杂度是 $ O(n) $ , 不需要对数组进行一次全排序就可以完成洗牌。

算法可以被描述为：

Let X1, X2…. XN be the set of N numbers to be shuffled.

  1. Set j to N
  2. Generate a random number R. (uniformly distributed between 0 and 1)
  3. Set k to (jR+1). k is now a random integer, between 1 and j.
  4. Exchange Xk and Xj
  5. Decrease j by 1.
  6. If j > 1, return to step 2.

用一个游标从头到尾遍历一遍，然后针对每一个新的元素，随机出这个区间的一个数，然后把当前元素和随机出来的元素进行替换，这样能达到在 $ O(N) $ 之内完成洗牌的任务。
