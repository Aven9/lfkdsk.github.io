Algorithms:Union Find

发表于 2017-03-15 | |

重新学习算法知识的第一节，Algorithms中的1.5节，并差集。

并差集可以判断一对对象的连通性（是否有等价关系），也被叫做动态连通性算法。并差集的重点在于怎么把所有的子节点划分成多个集合，如果两个点有连通性就将它们划分到同一个集合里面去，这样子当我们发现两个点是在一个集合中我们就能认为她们是可达的。

Tips： 等价关系：

  • 自反性
  • 对称性
  • 传递性

return value Function name Usage
UF(N) 生成0~N-1个点（我们可以把立体的并差集扁平化）

三种算法

QuickFind

和名字一样，这个算法的速度快在了Find上面，每次的find操作是O(1)的，但是本身是效率最差的一种。

并差集的目的就是通过某种标识符的方法将元素划分开，QuickFind这样子虽然每次的查找好像很快，但是其实每次的union要做一次O（n）的操作。

QuickUnion

QuickUnion的方法就高明很多，上面的重整标识符产生了很大的消耗，这样子我们就应该避免这个消耗，结成一棵树无疑是很好的解决问题的方法，只要两个节点有相同的根，那我们就认为他们在相同的集合。

初始化的时候，所有的标识符都指向自己，所以在搜索根节点的时候才能靠根节点的id灯不等于value来判断。在连接的时候我们也直接可以直接把一个的根付给另一个根。union不再是线性级别的运算啦。

Weight-QuickUnion

使用了树结构最大的问题就是树的纵深问题，树的高度变高之后，由下到上到根的线性消耗仍然不容小觑，这时我们采取一种策略，每当大树和小树合并的时候，我们总是把小树作为大树的子树，这样子我们就可以有效的缩减树的高度。

我们为每个节点都添加了权重值，即每个节点的子节点的层数，这样每次在合并树的时候都会进行改变。这样的树的层数就得到了有效的控制。

Tips1: 路径压缩

我们希望我们的多个节点都能直接连到根节点上，但是还不想要QuickUnion那种低效率。

每次find的时候都会进行路径压缩。

Tips2 : lgN

命题： N个触点，深度最多有 lgN 层

我们在算树的复杂程度的时候经常都是存在 lgN 这个复杂度，这里我们可以简单证明一下。

假设我们有两颗子树， T1 , T2 , 其中 T1 < T2 ,这样我们应该把 T1 连在 T2 的下面。

T2 > T1 那么 T2 至少也要比 T1 多一个节点，这样子这个合并过程，其实是一个节点翻倍的过程。

这样我们要到 N 个节点 就要翻 lgN 倍。
